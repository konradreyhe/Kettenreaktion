## r/webgames (Reddit)

**Title:** Kettenreaktion — A daily physics puzzle where you place ONE object and watch the chain reaction

**Body:**

Hey r/webgames!

I've been building **Kettenreaktion** (German for "chain reaction") — a daily physics puzzle game you play in your browser.

**How it works:**
- Every day there's a new puzzle (same for everyone worldwide)
- You place ONE object (ball, crate, weight, or domino) in a highlighted zone
- Physics takes over — your object triggers a chain reaction
- Goal: hit all the stars in as few attempts as possible
- Share your result as emoji text or a GIF replay

**What makes it different:**
- One puzzle per day — like Wordle, but physics
- 150 hand-crafted levels (5 months of daily content)
- Real physics simulation (Matter.js)
- Works on desktop and mobile (landscape)
- No login, no ads, no download — just play

**Looking for feedback on:**
- Is the difficulty curve right? (Starts easy, gets hard)
- Are the controls intuitive?
- How's the sharing/GIF export?
- Any bugs or weird behavior?

Play here: https://konradreyhe.github.io/Kettenreaktion/

Built with TypeScript, Phaser 3, and Matter.js. Happy to answer any technical questions too.

## Hacker News (Show HN)

**Title:** Show HN: Kettenreaktion — A daily physics puzzle game in the browser

**Body:**

I built a daily physics puzzle game called Kettenreaktion (chain reaction).

Each day there's one puzzle. You place a single object in a highlighted zone, and physics simulation (Matter.js) handles the rest. The goal is to trigger a chain reaction that hits all the target stars. Everyone worldwide gets the same puzzle, so you can compare results.

It's a browser game — TypeScript + Phaser 3 + Matter.js, no backend needed for the core game. Levels are baked into the build. GIF replay export runs in a Web Worker so it doesn't block the UI.

150 hand-designed levels covering 5 difficulty tiers. Works offline via service worker. No account, no ads.
